# 🚗 基于 CARLA 仿真器的交通标志检测与车辆控制

[![Python 3.7+](https://img.shields.io/badge/Python-3.7%2B-blue)](https://python.org)
[![CARLA](https://img.shields.io/badge/CARLA-0.9.13%2B-green)](https://carla.org)
[![YOLOv8](https://img.shields.io/badge/YOLO-v8-red)](https://ultralytics.com)

在 **CARLA 仿真器** 中实现自动驾驶行为模拟，使用 **YOLOv8** 实时检测交通标志（停车标志、限速标志）和 **障碍物**，车辆自动调整行驶行为。支持 **碰撞预警与自动避让**，并提供 **Pygame 画面显示** 和 **控制台实时仪表盘** 两种监控方式。

---

## ✨ 功能特性

### 基础功能
- **车道保持** — 基于路径点（Waypoint）的自动转向控制
- **交通标志检测** — YOLOv8 实时检测停车标志、限速标志
- **交通信号灯响应** — 红灯自动制动
- **动态交通流** — 随机生成 10 辆 NPC 车辆，开启自动驾驶
- **资源自动清理** — 仿真结束后销毁所有实体，防止内存泄漏

### 新增创新功能

| 功能 | 说明 |
|------|------|
| 🖼️ **检测结果可视化** | 在驾驶员画面上绘制彩色边界框，不同类别不同颜色 |
| 📏 **单目距离估计** | 基于边界框大小估算物体与本车距离 |
| ⚠️ **分级碰撞预警** | 40m→25m→15m→8m 四级响应，自动减速或紧急制动 |
| 📊 **增强 HUD** | 右上角检测统计、右下角最近障碍物距离、中央碰撞警告 |
| 💻 **控制台仪表盘** | 不依赖 Pygame，终端实时刷新状态面板 |

### 额外需求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| **CARLA 仿真器** | ≥ 0.9.13 | [下载地址](https://github.com/carla-simulator/carla/releases) |

## ▶️ 运行指南

### 1️⃣ 启动 CARLA 仿真器

**Windows:**
```bash
CarlaUE4.exe
```

**Linux:**
```bash
./CarlaUE4.sh
```

等待仿真器加载完成，看到城市画面即可。

### 2️⃣ 运行主程序

```bash
python main.py
```

**使用虚拟环境（推荐）：**
```bash
.venv\Scripts\python main.py    # Windows
source .venv/bin/activate && python main.py  # Linux
```

### 3️⃣ 运行效果

- 弹出一个 **Pygame 窗口**，显示驾驶员视角和检测结果
- **控制台** 实时刷新状态仪表盘
- 仿真默认运行 **2 分钟** 后自动停止
- 按 **ESC** 或关闭窗口可提前退出

```
│                    └────────────────────┘              │
└─────────────────────────────────────────────────────────┘
```

### 核心流程

1. **图像采集** — CARLA 的 RGB 摄像头每帧捕获图像
2. **YOLOv8 推理** — 检测图像中的交通标志、车辆、行人等
3. **距离估计** — 基于边界框大小估算物体与本车距离
4. **碰撞预警** — 根据距离分级响应（40m→预警、25m→减速、15m→强减速、8m→紧急制动）
5. **车辆控制** — 综合交通标志和障碍物信息，控制油门/刹车/转向
6. **可视化** — Pygame 显示带边界框的驾驶员视角 + 控制台仪表盘

## 🎯 检测类别与颜色

| 类别 | 颜色 | 说明 |
|------|------|------|
| `person` | 🔴 红色 | 行人 |
| `car` / `truck` / `bus` | 🔵 蓝色 | 车辆障碍物 |
| `bicycle` / `motorcycle` | 🟠 橙色 | 非机动车 |
| `traffic light` | 🟡 黄色 | 交通灯 |
| `stop sign` | 🟣 紫色 | 停车标志 |
| 其他 | 🟢 绿色 | 默认 |

---

## 📊 碰撞预警等级

| 距离 | 响应 | 显示 |
|------|------|------|
| < 8m | 🚨 **紧急制动**，刹车 100% | 红色闪烁大字 |
| 8m - 15m | ⚠️ **强力减速**，刹车 80% | 红色警告 |
| 15m - 25m | ⚠️ **轻微减速**，刹车 20% | 黄色提示 |
| 25m - 40m | ℹ️ **提前预警**，不制动 | 黄色文字提示 |
| > 40m | 正常行驶 | 绿色正常 |

## 🔧 配置说明

可在 `main.py` 中调整以下参数：

| 参数 | 位置 | 说明 | 默认值 |
|------|------|------|--------|
| 仿真时长 | `if elapsed > 120` | 运行秒数 | 120s |
| 检测阈值 | `conf=0.5` | 置信度阈值 | 0.5 |
| 摄像头分辨率 | `image_size_x/y` | 画面尺寸 | 800x600 |
| FOV | `fov` | 摄像头视场角 | 90° |
| NPC 数量 | `for _ in range(10)` | 交通流数量 | 10 |
| 紧急制动距离 | `distance < 8.0` | 紧急触发阈值 | 8m |
| 预警距离 | `distance < 40.0` | 预警触发阈值 | 40m |

## 🧼 资源清理

仿真结束后会自动销毁所有实体（车辆、传感器、交通标志等），确保 `finally` 块中的清理代码执行，避免内存泄漏。

## 📌 注意事项

1. **Python 版本** — CARLA 0.9.14 的 egg 文件依赖 Python 3.7，请使用虚拟环境运行
2. **模型精度** — 默认使用 `yolov8n.pt`（轻量），可替换为 `yolov8s.pt`、`yolov8m.pt` 提高精度
3. **场景资源** — 确保 CARLA 地图中包含脚本引用的交通标志模型资源
4. **YOLO 类别** — 检测依赖 COCO 数据集的预训练类别，如需精准分类交通标志，建议微调模型
5. **显示问题** — Pygame 画面出现裂痕/撕裂时，可调整 `sensor_tick` 和 `gamma` 参数
