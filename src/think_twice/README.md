# ThinkTwice 自动驾驶算法模块开发文档
## 一、项目概述
### 1.1 项目背景
自动驾驶技术融合计算机视觉、深度学习、传感器融合、路径规划等多项技术，是人工智能领域核心落地赛道。感知模块作为自动驾驶系统的环境感知单元，负责识别车辆、行人、车道线、交通标识等周边目标，是决策、规划、控制模块正常运行的前置基础。

ThinkTwice 算法模块面向自动驾驶3D多模态感知场景开发，基于主流深度学习框架搭建标准化工程体系，覆盖数据加载、模型推理、后处理、结果输出全链路。本模块严格遵循团队 `nn` 项目统一目录规范，代码完全解耦，可无缝集成至主工程；同时针对性解决嵌套Git仓库、路径异常、推理卡顿、版本污染等工程痛点，保障算法稳定运行与代码版本合规管理。

### 1.2 模块功能介绍
本模块以多传感器融合3D感知为核心，具备以下完整能力：
1. 兼容图像、激光点云多模态车载数据读取与标准化预处理；
2. 支持自研/开源3D检测模型加载，完成障碍物分类、3D空间定位、尺寸估算推理；
3. 内置推理结果后处理逻辑，过滤低置信度无效框、校正目标三维坐标；
4. 统一结构化输出感知结果，为上层决策规划模块提供标准化输入；
5. 集成分级日志、运行状态监控、全局异常捕获工具，简化本地调试与线上问题排查。

### 1.3 项目目标
1. 模块化工程独立开发，仅新增专属文件夹，不修改 `nn` 项目原有任何源码；
2. 彻底清除目录内嵌套Git仓库，提交文件均为原生业务代码，规避Git子模块提交异常；
3. 打通数据-模型-后处理-输出完整自动化流程，代码无语法、运行时报错；
4. 分层低耦合架构设计，支持快速替换模型、切换数据集、迭代新增感知功能；
5. 修复路径解析失败、推理资源占用过高、第三方依赖缺失等运行故障，保障长时间稳定推理。

### 2.3 子目录功能说明
1. **config**：集中管理模型路径、输入分辨率、置信度阈值、数据集路径等可变参数，修改配置无需改动业务推理代码；
2. **dataset**：负责车载图像、激光点云原始数据读取、格式转换、归一化、数据增广，解决多传感器数据格式不兼容问题；
3. **model**：包含网络骨干、3D障碍物检测、推理后处理全套核心算法，实现目标检测、坐标校正、低置信框过滤；
4. **utils**：封装路径校验、日志打印、全局异常捕获通用函数，提升代码复用性，统一日志输出格式；
5. **open_loop_training**：存放算法训练、离线推理相关第三方组件，已全部清除内部`.git`文件夹，无嵌套子仓库；
6. **main.py**：模块启动入口，串联数据加载、模型初始化、逐帧推理、结果输出完整业务流程。

## 三、完整代码实现与说明
### 3.1 模块入口 main.py
```python
import os
import sys
import time
from config.model_cfg import MODEL_CONFIG
from config.data_cfg import DATA_CONFIG
from dataset.loader import get_dataloader
from model.detector import Detector
from utils.logger import create_logger

# 全局日志初始化
logger = create_logger()

def main():
    # 1. 全局绝对路径校验，规避相对路径失效问题
    root_path = os.path.dirname(os.path.abspath(__file__))
    sys.path.append(root_path)
    logger.info(f"ThinkTwice模块启动成功，当前模块根路径：{root_path}")

    # 2. 加载多模态车载数据集
    try:
        dataloader = get_dataloader(DATA_CONFIG)
        logger.info("多模态数据集加载完成，就绪等待推理")
    except Exception as e:
        logger.error(f"数据集加载异常终止：{str(e)}")
        return

    # 3. 初始化3D感知检测模型
    try:
        detector = Detector(MODEL_CONFIG)
        logger.info("3D多模态感知模型初始化完成")
    except Exception as e:
        logger.error(f"模型初始化失败：{str(e)}")
        return

    # 4. 自动驾驶逐帧推理主循环
    logger.info("启动自动驾驶感知推理流程......")
    start_time = time.time()
    for idx, data in enumerate(dataloader):
        # 执行单帧3D检测推理
        result = detector.infer(data)
        # 打印单帧感知输出：目标类别、三维坐标、置信度
        logger.info(f"第{idx+1}帧感知推理结果：{result}")

        # 限制推理间隔，控制硬件资源占用，避免卡顿发热
        time.sleep(0.05)

    total_time = time.time() - start_time
    logger.info(f"全部数据推理完成，总运行时长：{total_time:.2f} s")

if __name__ == "__main__":
    try:
        main()
    except KeyboardInterrupt:
        logger.info("程序收到手动终止信号，正常退出")
    except Exception as e:
        logger.error(f"程序全局运行异常：{str(e)}")
```

### 3.2 核心模块划分说明
1. **config 配置模块**
    将模型参数、数据集参数解耦分离，统一管理推理阈值、文件路径、输入尺寸，支持一键切换模型与测试数据集，适配多场景快速调参。
2. **dataset 数据模块**
    适配自动驾驶图像、激光雷达多模态原始数据，完成数据清洗、归一化、坐标系转换，统一输入格式，解决各类传感器数据读取、解析失败问题。
3. **model 算法模块**
    承载3D感知网络前向推理、目标框筛选、三维坐标校正、目标分类核心逻辑；配套后处理模块剔除低置信干扰目标，提升环境感知精准度，是模块核心算法载体。
4. **utils 工具模块**
    封装标准化日志、路径合法性校验、全流程异常捕获工具，统一日志输出规范，快速定位报错节点，降低线下调试成本。
5. **主程序与全局异常处理**
    串联数据加载、模型初始化、逐帧推理三大核心流程，关键节点增加独立异常捕获，单一环节报错不会导致整体程序崩溃，大幅提升模块运行健壮性。

## 四、开发常见问题与解决方案
### 4.1 问题1：Git识别目录为嵌套子模块（create mode 160000）
**现象**：提交代码时提示存在embedded git repository，文件提交类型为`160000`，仅提交仓库引用，无真实代码文件，代码审核驳回。
**原因**：`think_twice`内部`mmcv`、`mmdetection3d`文件夹残留`.git`目录，Git判定为嵌套独立子仓库。
**解决方案**
1. 批量递归清理模块内所有嵌套Git文件：
```bash
find src/think_twice -name ".git" -exec rm -rf {} \;
```
2. 清除Git缓存中的子模块标记：
```bash
git rm -r -f --cached src/think_twice/open_loop_training/mmcv
git rm -r -f --cached src/think_twice/open_loop_training/mmdetection3d
```
3. 重新添加全部源码并提交，确保提交文件均为原生代码。

### 4.2 问题2：文件路径解析失败，提示No such file or directory
**现象**：程序启动时报目录/文件不存在，无法加载模型权重、数据集文件。
**原因**：代码使用相对路径，切换终端工作目录后路径失效；文件夹名称大小写、拼写错误。
**解决方案**
1. 全局统一使用`os.path.abspath()`转换为绝对路径，消除工作目录影响；
2. 严格对齐代码内路径名称与本地文件夹命名；
3. 在工具函数中新增路径预校验逻辑，程序启动前提前检测文件/目录是否存在并抛出提示。

### 4.3 问题3：推理卡顿、CPU/GPU占用率过高
**现象**：循环推理无间隔限制，硬件满载运行，设备发热、画面响应缓慢。
**原因**：推理循环全速无休眠，持续抢占系统计算资源。
**解决方案**：在逐帧推理循环中增加`time.sleep()`休眠逻辑，控制单帧运行间隔，平衡推理速度与硬件资源占用。

### 4.4 问题4：直接基于main主分支提交PR
**现象**：主分支代码被污染，代码历史断裂，合并产生大量版本冲突，无法完成合入。
**原因**：`main`为项目官方稳定分支，禁止直接在主分支编写、提交功能代码。
**解决方案**：基于上游`upstream/main`新建独立功能分支（如`final_pr`），全部开发、代码提交、PR合并操作仅在独立分支执行，保护主分支代码稳定。

### 4.5 问题5：Git切换分支提示Directory not empty
**现象**：执行`git checkout`切换官方主分支时，提示无法删除`src/think_twice`目录。
**原因**：官方主分支不存在该模块目录，本地存在未被Git追踪管控的残留文件。
**解决方案**：切换分支前备份模块代码，再清空本地`think_twice`目录后执行分支切换。

## 五、功能扩展方向
### 5.1 模型迭代与快速替换
修改`config/model_cfg.py`内模型权重路径、推理超参，无需改动推理主逻辑，即可切换不同版本3D感知模型，快速完成算法迭代对比。

### 5.2 新增多类传感器适配
扩展`dataset`模块预处理流水线，新增毫米波雷达、多路摄像头数据解析逻辑，实现图像、点云、雷达多传感器深度融合感知。

### 5.3 上层决策模块对接
统一标准化感知结果输出接口，向外输出障碍物三维位置、类别、运动速度等结构化信息，实现与自动驾驶决策、路径规划模块端到端联调。

### 5.4 感知结果可视化功能
基于OpenCV拓展可视化工具，在图像、点云画面绘制3D检测框、目标坐标，直观展示算法感知效果，方便线下效果调试。

### 5.5 模型完整训练流程完善
迭代优化`open_loop_training`目录代码，补充数据集训练流水线、自定义损失函数、精度评估指标，支持从零训练专属自动驾驶感知模型。
