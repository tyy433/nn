# 基于 PPO + MuJoCo 的人形机器人强化学习站立训练系统

## 1 项目背景与研究动机

### 1.1 行业发展现状
人形机器人的运动控制是机器人学领域的核心挑战之一。传统基于模型的控制方法（如 ZMP、LQR、MPC）需要精确的动力学模型和大量的手工调参，难以应对复杂多变的环境。近年来，强化学习凭借其无需精确建模、自主学习策略的特点，成为人形机器人控制的热门方向。波士顿动力、特斯拉 Optimus、小米 CyberOne 等前沿项目均在不同程度上采用了强化学习技术。

### 1.2 研究动机与目标
本项目基于 MuJoCo 物理仿真引擎和 Gymnasium 强化学习接口，构建一个完整的人形机器人站立训练系统。核心目标包括：
- 构建一个具有 11 个自由度（颈、肩、肘、髋、膝、踝）的力矩控制人形机器人模型；
- 利用 PPO（Proximal Policy Optimization）算法训练机器人从随机初始状态学习稳定站立；
- 通过精心设计的奖励函数（存活奖励、高度奖励、直立奖励），引导机器人学会保持平衡；
- 搭建模块化、可扩展的仿真实验平台，支持训练、测试、继续训练等多种运行模式。

## 2 整体技术架构

系统采用分层架构设计，各模块通过标准化接口通信：

| 层级 | 核心功能 | 关键模块 | 技术要点 |
|------|----------|----------|----------|
| **物理仿真层** | MuJoCo 物理引擎，提供刚体动力学计算 | `humanoid_rl.xml` 模型文件、力矩控制执行器 | 11 个铰链关节、力矩控制范围 [-50, 50] N·m、时间步长 0.005s |
| **环境接口层** | Gymnasium Env 标准接口封装，提供状态观测、动作执行、奖励计算 | `HumanoidRLEnv` 类 | 状态向量 35 维（根节点位姿+速度 + 关节位置+速度）、动作空间 11 维力矩控制 |
| **智能体层** | PPO 算法实现策略学习 | Stable-Baselines3 PPO 实现 | MLP 策略网络、Actor-Critic 架构、PPO-Clipped 目标函数 |
| **训练调度层** | 训练流程控制、模型管理、评估回调 | `StatsCallback`、`EvalCallback` | 200 万步训练、每 5000 步打印统计、每 10000 步评估保存最优模型 |

## 3 核心理论基础

### 3.1 PPO 算法原理
PPO 是一种基于策略梯度的强化学习算法，通过裁剪目标函数限制策略更新幅度，保证训练稳定性：

- **Actor-Critic 架构**：Actor（策略网络）输出动作分布，Critic（价值网络）估计状态价值；
- **PPO-Clip 目标函数**：`L = min(rA, clip(r, 1-ε, 1+ε)A)`，其中 r 为新旧策略概率比，A 为优势函数，ε 为裁剪范围（本项目设为 0.2）；
- **GAE（广义优势估计）**：平衡偏差和方差，本项目 λ=0.97；
- **多轮更新**：每采集一批数据，对网络进行 10 轮（n_epochs）更新，提高样本效率。

### 3.2 力矩控制模型
机器人的 11 个关节均通过力矩马达（Motor）驱动，控制信号为期望力矩值：

| 关节 | 控制范围 (N·m) | 阻尼系数 | 关节限位 |
|------|---------------|---------|---------|
| neck（颈） | [-20, 20] | 0.5 | ±0.5 rad |
| l_shoulder / r_shoulder（肩） | [-30, 30] | 0.5 | ±1.5 rad |
| l_elbow / r_elbow（肘） | [-15, 15] | 0.3 | -1.5 ~ 0 rad |
| l_hip / r_hip（髋） | [-50, 50] | 1.0 | ±1.0 rad |
| l_knee / r_knee（膝） | [-50, 50] | 1.0 | 0 ~ 2.0 rad |
| l_ankle / r_ankle（踝） | [-30, 30] | 0.5 | ±0.5 rad |

### 3.3 奖励工程
奖励函数是强化学习成功的关键，本项目设计了三维复合奖励：

- **存活奖励**（+0.1）：每步给予微小奖励，鼓励机器人保持存活；
- **高度奖励**（+0.2 × torso_z）：身体躯干越高，奖励越大，引导站立而非趴下；
- **直立奖励**（+0.1 × quat_w）：躯干四元数 w 分量越大（越接近直立），奖励越大。

### 3.4 终止条件
训练在以下情况提前终止（episode done），避免无意义的学习：
- **跌倒终止**：躯干高度 < 0.5m；
- **倾斜终止**：躯干四元数 w 分量 < 0.2（严重倾斜）；
- **越界终止**：水平位移超过 3m（即 sqrt(x² + y²) > 3）。

## 4 详细方案设计

### 4.1 状态空间设计

| 状态分量 | 维度 | 含义 |
|----------|------|------|
| root_qpos | 7 | 根节点位置（x, y, z）+ 姿态四元数（w, x, y, z） |
| joint_pos | 11 | 11 个关节的角度值 |
| root_qvel | 6 | 根节点线速度（vx, vy, vz）+ 角速度（ωx, ωy, ωz） |
| joint_vel | 11 | 11 个关节的角速度 |

总计 7 + 11 + 6 + 11 = **35 维**状态向量。

### 4.2 动作空间设计
- 维度：11（对应 11 个关节力矩）；
- 范围：[-1, 1]（归一化），实际力矩 = action × 10 N·m；
- 裁剪：最终输出被裁剪到 [-50, 50] N·m，与 MuJoCo 模型执行器的 ctrlrange 一致。

### 4.3 PPO 超参数配置

| 参数 | 值 | 说明 |
|------|-----|------|
| learning_rate | 1e-4 | 策略和价值网络学习率 |
| n_steps | 2048 | 每次采集的交互步数 |
| batch_size | 128 | 小批量更新大小 |
| n_epochs | 10 | 每批数据的更新轮数 |
| gamma | 0.995 | 折扣因子（接近 1，关注远期回报） |
| gae_lambda | 0.97 | GAE 参数 |
| clip_range | 0.2 | PPO 裁剪范围 |
| ent_coef | 0.01 | 熵正则化系数（鼓励探索） |
| total_timesteps | 2,000,000 | 总训练步数 |
| device | cpu | 计算设备（可改为 cuda） |

定义了人形机器人的完整物理模型：
- **身体部件**：躯干（torso）、头部（head）、左右臂（肩+肘）、左右腿（髋+膝+踝）；

- **执行器**：11 个力矩马达（motor），配置了控制范围和齿轮比；
- **环境**：地面平面（plane），重力加速度 9.81 m/s²。

## 6 运行环境与部署步骤

### 6.1 环境依赖

- Python 3.8+
- MuJoCo >= 2.3.7
- Gymnasium >= 0.28.0
- Stable-Baselines3 >= 2.0.0
- PyTorch >= 2.0.0
- NumPy >= 1.24.0

### 6.2 安装步骤

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 验证 MuJoCo 安装
python -c "import mujoco; print(mujoco.__version__)"
```

### 6.3 启动训练

```bash
# 进入项目目录
cd src/robotic_arm_control

# 启动训练程序
python humanoid_rl_train.py

# 交互菜单:
#   1 - 训练新模型
#   2 - 测试现有模型
#   3 - 继续训练（在已有模型基础上继续 100 万步）
```

### 6.4 测试模型

```bash
# 方式一：通过菜单选择 "2. 测试现有模型"
python humanoid_rl_train.py

# 方式二：直接运行手动控制演示
python humanoid_sim.py
```

### 7.4 常见问题与解决

| 问题现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 机器人快速倒地 | 力矩过大导致不稳定 | 调整 `torque = action * scale` 中的 scale 值（默认 10），降低力矩缩放因子 |
| 奖励不收敛 | 奖励权重不合理 | 调整 `_compute_reward()` 中各奖励项的系数 |
| 渲染窗口无响应 | MuJoCo viewer 未正确启动 | 检查 MuJoCo 版本（>= 2.3.7），或使用 `render_mode=None` 关闭渲染 |
| 模型加载失败 | 路径问题或版本不兼容 | 确认模型文件存在于 `./models/` 目录下，检查 SB3 版本 |
| 训练速度慢 | CPU 计算瓶颈 | 将 `device="cpu"` 改为 `device="cuda"`（需 NVIDIA GPU） |

## 8 后续拓展方向

1. **行走训练**：在站立基础上引入前进速度奖励，训练机器人迈步行走；
2. **多任务学习**：同时训练站立、行走、转向等多种技能，实现任务切换；
3. **Sim-to-Real**：添加域随机化（domain randomization），提高模型从仿真到真实机器人的迁移能力；
4. **算法对比**：集成 SAC、TD3 等算法，对比不同算法在力矩控制任务上的性能；
5. **视觉输入**：引入视觉传感器（RGB/深度图），训练基于视觉的端到端站立策略；
6. **鲁棒性增强**：添加随机外力扰动（推力、风等），训练机器人抵抗外部干扰的能力。
